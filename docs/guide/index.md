# 简介

Industry Vis 是一个基于 Tauri 的工业数据可视化系统，专为读取、处理和展示工业运行数据而设计。

::: tip 适用场景
适合需要查看和分析工业历史数据的工程师、运维人员和数据分析师。
:::

## 主要功能

### 数据连接

- **SQL Server 连接** — 支持连接 SQL Server 数据库，读取工业历史数据
- **标签模糊搜索** — 从 TagDatabase 表中搜索标签，支持中文

### 数据展示

- **时间范围选择** — 预设时间范围（实时/1小时/班次/今日/昨日/本周）和自定义时间范围
- **标签分组管理** — 创建标签分组（最多20个标签/组），便于批量查询
- **折线图** — 多系列折线图，支持缩放平移和数据点提示

### 数据处理

- **异常值剔除** — 使用 3σ 法则自动识别并移除离群点
- **时间序列重采样** — 按指定间隔聚合数据，减少数据量
- **平滑滤波** — 移动平均滤波，平滑噪声数据
- **自动降采样** — 超过5000点时自动降采样，优化前端渲染

### 其他功能

- **CSV 导出** — 支持导出筛选后的数据为 CSV 文件
- **配置持久化** — 便携模式和安装模式自动选择
- **主题切换** — 支持明暗主题

## 数据模型

### 历史表

系统默认查询 `控制器数据库` 的 `历史表`：

| 列名 | 类型 | 说明 |
|------|------|------|
| `DateTime` | datetime | 时间戳 |
| `TagName` | nvarchar(50) | 标签名称 |
| `TagVal` | real | 数值 |
| `TagQuality` | nchar(10) | 质量标志 |

### TagDatabase（标签搜索）

| 列名 | 类型 | 说明 |
|------|------|------|
| `TagName` | nvarchar(50) | 标签名称 |

::: info 自定义数据库结构
如果你的数据库结构与默认不同，可以通过 [Schema Profile](/guide/schema-profile) 功能进行适配。
:::
